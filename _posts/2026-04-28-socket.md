类型sockaddr：
就是一个通用的地址结构体，经常使用的是类似的结构体sockaddr_in(IPv4)和sockaddr_in6(IPv6)，不是继承关系，但是使用(类型)可以转换

socket相关函数：
/**
 * @brief 创建通信端点（套接字）
 * @param domain 协议族/地址族
 *     - AF_INET: IPv4协议
 *     - AF_INET6: IPv6协议  
 * @param type 套接字类型
 *     - SOCK_STREAM: 面向连接的TCP套接字（可靠，双向字节流）
 *     - SOCK_DGRAM: 无连接的UDP套接字（不可靠，数据报文）
 * @param protocol 具体协议类型,通常设为0,由前两个参数自动选择
 * @return 成功返回非负文件描述符，失败返回-1
 */
int socket(int domain, int type, int protocol);
/**
 * @brief 将套接字与本地地址和端口绑定
 * @param sockfd socket()返回的文件描述符
 * @param addr 指向地址结构的通用指针，实际应为sockaddr_in或sockaddr_in6
 * @param addrlen 地址结构的实际长度（字节数）
 * @return 成功返回0，失败返回-1
 */
int bind(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
/**
 * @brief 将主动套接字转换为被动监听套接字
 * @param sockfd 已成功bind()的TCP套接字描述符
 * @param backlog 已完成连接队列的最大长度
 *     - 表示系统可同时接受的连接数
 *     - 典型值：SOMAXCONN（系统最大）或5-10
 * @return 成功返回0，失败返回-1
 */
int listen(int sockfd, int backlog);
